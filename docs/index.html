What you need

  • 01
    Python 3.10+
    Check: python3 --version
  • 02
    Claude Code CLI
    npm install -g @anthropic-ai/claude-code
  • 03
    cloudflared
    macOS: brew install cloudflare/cloudflare/cloudflared  ·  Windows: winget install Cloudflare.cloudflared
  • 04
    A Google account
    For Drive API access
  • 05
    A Plaud device with Google Drive sync on
    Transcripts must auto-sync to a Drive folder

Installation

1

Clone the repo

Copy$ git clone https://github.com/OllieB1943/plaude-code.git
$ cd plaude-code
2

Install Python deps

Copy$ pip install -r requirements.txt
3

Run the setup wizard

Handles Google OAuth, folder selection, and config in one shot.

Copy$ python setup.py
4

Enable Plaud → Drive sync

In the Plaud app, turn on Google Drive sync and point it to the same folder you configured above. Check the Plaud help docs for where the setting lives.

5

Start the bridge

Copy$ ./start.sh

How it works

🎙
Record on Plaud Speak your instructions naturally
→
💬
Say "send to claude" Trigger phrase detected automatically
→
⚡️
Claude executes CLAUDE.md written, CLI launched

Trigger phrases

Any of these in your recording will kick it off. Case-insensitive.

send to claude send this to claude claude should claude execute claude fix claude finish claude do this want this to go to claude go to claude for claude claude code this claude handle claude [any verb] →

The catch-all matches anything starting with "claude" followed by a word — "claude refactor this", "claude clean up the tests", etc.

Troubleshooting

cloudflared not found
macOS: brew install cloudflare/cloudflare/cloudflared
Windows: winget install Cloudflare.cloudflared

Then re-run ./start.sh.
Google auth failed
Delete token.json and re-run python setup.py. Make sure you enabled the Google Drive API in the Google Cloud Console and that your OAuth credentials are for a Desktop app, not a web app.
Project directory not detected
The bridge looks for your folder names inside the transcript. If the project name isn't mentioned, try saying "working in my-project" or "in the api-service repo".

If nothing matches, a fallback CLAUDE.md drops to ~/Desktop/plaude-unmatched/ — just move it to the right project and run claude.
claude command not found
Install the CLI: npm install -g @anthropic-ai/claude-code

Verify with which claude. If it's still missing, set the full path in config.json under the claude_cmd key.
