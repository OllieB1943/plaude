4

Connect Plaud to Google Drive

This is how Plaude receives your transcripts. Follow these steps in the Plaud mobile app:

  1. Open the Plaud app on your phone
  2. Tap your profile icon or the Settings gear (top corner of the home screen)
  3. Find Cloud Sync, Backup, or Integrations
  4. Tap Google Drive and sign in with the same Google account you used during setup
  5. Plaud will create a folder in your Drive тАФ usually called Plaud or PLAUD
  6. Open Google Drive on your computer and navigate to that folder
  7. Copy the folder ID from the URL bar:
https://drive.google.com/drive/folders/THIS_PART_IS_YOUR_FOLDER_ID

Paste this ID when python3 setup.py asks for your Drive folder. Plaude will watch this folder for new transcripts.

Tip: Make sure sync is set to automatic in the Plaud app so transcripts upload as soon as a recording finishes тАФ no manual action needed.

Setting up Claude Cowork with Google Drive
Use the native Google Drive connector тАФ no local inbox needed
1
Open the Claude desktop app and start a new Cowork session
In the Claude desktop app, click Cowork in the sidebar and start a new session. Give it a name like "Plaude automation".
2
Add the Google Drive connector
In the Cowork session, click + Add connector тЖТ choose Google Drive.
Sign in with the same Google account your Plaud syncs to.
Select the Plaud folder (the one whose ID you copied during setup).
3
Give Cowork its standing instructions
Paste this as the session's initial message тАФ Cowork will follow it automatically every time a new transcript lands in the folder:
Watch my Google Drive Plaud folder for new files. Whenever a new transcript appears:

1. Read the full transcript text
2. Check if it contains a trigger phrase like "send to claude", "claude fix", "claude draft", "claude plan", or anything starting with "claude [verb]"
3. If a trigger is found, identify the project being referred to from the transcript text
4. Execute the instructions in the transcript fully and completely тАФ write code, draft documents, do research, or whatever is asked
5. When done, write a brief summary of everything you did

If no trigger phrase is found, ignore the file.
Stay running and repeat this for every new file that appears.
4
Leave Cowork running
That's it. Cowork stays open in the background. Every time Plaud syncs a new transcript to Drive, Cowork picks it up, checks for a trigger, and acts тАФ with no extra steps from you.

With this setup, you don't need to run ./start.sh at all тАФ Cowork handles everything end-to-end.
Tip: If you want both Cowork and Claude Code CLI to trigger, set "launcher": "both" in config.json and run ./start.sh alongside Cowork.

Trigger phrases

Say any of these in your recording. Works for code, writing, planning тАФ anything Claude can do.

send to claude send this to claude claude should claude handle claude fix claude draft claude summarise claude plan claude do this want this to go to claude go to claude for claude claude [anything] тЖТ

The catch-all matches any phrase starting with "claude" followed by a word тАФ "claude write me an email", "claude make a plan", "claude research this".

Troubleshooting

cloudflared not found
macOS: brew install cloudflare/cloudflare/cloudflared
Windows: winget install Cloudflare.cloudflared

Then re-run ./start.sh.
Google auth failed
Delete token.json and re-run python setup.py. Make sure you enabled the Google Drive API in Google Cloud Console and that your OAuth credentials are for a Desktop app, not a web app.
Project directory not detected
Plaude scans the transcript for folder names that exist in your projects_root. If the project name isn't mentioned, try saying "working in my-project" or "in the api-service repo".

If nothing matches, a fallback file drops to ~/Desktop/plaude-unmatched/.
claude command not found
Install the CLI: npm install -g @anthropic-ai/claude-code

Verify with which claude. Or switch to the Cowork launcher in config.json to use the desktop app instead.
